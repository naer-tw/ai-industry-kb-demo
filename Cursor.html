Cursor

一句話定義

Cursor 是由 Anysphere 公司開發的 AI 原生程式碼編輯器（fork 自 VS Code），以 AI agent 自動寫程式、改 bug、跑多任務為核心；在 AI 供應鏈屬下游「應用層」，是 2024–2026 年成長最快的 B2B 軟體公司，靠企業開發者市場在約三年內衝上 20 億美元年化營收，被視為 AI coding 賽道的明星指標。

在 AI 供應鏈的位置與角色

Cursor 位居 AI 供應鏈的下游「應用／工具層」，是把上游算力與中游基礎模型「封裝成開發者生產力工具」的代表：

  • 對中游（模型層）的依賴：Cursor 本身長期不自有前沿模型，核心能力建構在 Anthropic 的 Claude 系列 與 OpenAI 的 GPT 系列 API 之上；同時自行訓練 in-house 模型 Composer（初代 2025-10 推出，稱「首個 agentic coding model」，係 Cursor 自行訓練）。惟 Composer 2（2026-03）與 Composer 2.5（2026-05-18）並非從零自研，而是建構在中國 Moonshot AI（月之暗面）開源的 Kimi K2.5 checkpoint 之上做後訓練（continued pretraining + RL）；Composer 2.5 宣稱以約 1/10 token 成本逼近 Claude／GPT 前沿 coding 水準（來源 9、11、12，Cursor 官方部落格／VentureBeat／The Decoder）。
  • 對上游（算力）的連結：透過與 SpaceX／xAI 合作取得算力，使用數萬顆 xAI 晶片訓練模型、接入 Colossus 超級電腦（來源 6）；投資方含 NVIDIA（同時是其大客戶）。
  • 產品形態：fork 自微軟 VS Code 的 AI 原生編輯器，2025-10 推出 Cursor 2.0 支援多 agent 平行執行、Bugbot（2025-07）做 PR 自動程式碼審查，屬 AI Agent與自動化 在軟體開發領域的旗艦落地。
  • 競爭格局：與 GitHub Copilot、Windsurf（2025 被 OpenAI 以約 30 億美元收購相關資產）、Claude Code 直接競爭；上游模型供應商同時是潛在競爭者（model 廠自推 coding 工具），形成「既依賴又被夾擊」的位置。

危機 / 變化 / 風險

  • 模型依賴與供應商箝制：核心體驗高度倚賴 Anthropic Claude 與 OpenAI GPT。2025 年 Anthropic 對第三方「冒用 Claude Code 客戶端」與經由 Cursor 訓練競品的行為祭出技術封鎖、調整計費分層，使 Cursor 用戶暴露於「上游一改政策即受衝擊」的風險（來源 10，VentureBeat；MindStudio 報導）。
  • 上游即競爭者：Claude Code、GitHub Copilot、OpenAI（收 Windsurf）等模型／平台廠自推 coding agent，與 Cursor 正面競爭，壓縮其差異化空間。
  • 估值與泡沫疑慮：估值從 99 億（2025-06）暴衝至 293 億（2025-11），2026-04 傳募資估值約 500 億美元，ARR 雖快速成長但獲利路徑、留存率與 AI coding 賽道泡沫化為市場關注點；該 500 億美元一輪最終遭 SpaceX 收購要約搶先而中止、從未完成交割（來源 5、來源 6）。
  • 被收購不確定性：SpaceX 的 600 億美元收購為「選擇權」而非確定交易，與 SpaceX IPO 時程綁定；若 SpaceX 改付 100 億美元或交易告吹，公司獨立性與戰略方向將生變（來源 6、7）。
  • 自研模型落差：Composer 雖主打低成本，但官方與媒體均承認 Cursor／xAI 自有模型仍難全面匹敵 Anthropic、OpenAI 的前沿模型（來源 6）。
