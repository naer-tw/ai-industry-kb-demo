CSV資料分析

一句話定義（這個可交付物是什麼）

使用者上傳一份結構化資料檔（CSV/Excel）並用自然語言提問（如「各區營收差異？哪個變數最相關？」），AI 自行寫出可執行的 Python/SQL 程式碼，跑出描述統計、產生圖表，並回一句結論。屬「對話式資料分析 / Code Interpreter」類能力，是目前主流大型模型已商品化的核心功能之一。

難度評分（綜合 3/10：技術 3、落地 4）— 理由

  • 技術 3（可用但需把關）：單步、乾淨的描述統計與繪圖任務模型已做得很好——InfiAgent-DABench（257 題、52 個 CSV）GPT-4o 達 75.9%（來源 2），DABStep 簡單題目開源 Llama 70B 零樣本即可超過 90%（來源 1）。但一旦變成多步、需跨表/讀文件、做正確口徑的「真實分析」，準確率斷崖式下滑：DABStep 難題最佳推理模型 o3-mini 僅 16%、Claude 3.5 Sonnet 12%、DeepSeek V3 6%（來源 1）。技術成熟但「會算對」與「會算對難題」差距大。
  • 落地 4（已大規模產品化、但可靠性需人工把關）：ChatGPT Advanced Data Analysis、Claude、Excel Copilot 均已 GA 並廣泛部署（來源 3、5）；落地障礙不在「能不能做」而在「敢不敢信」——Excel Copilot 同一輸入重算結果可能不同、且會產生看似合理的錯誤公式，微軟與專家明確建議財務預測、迴歸、統計建模等高風險場景「跳過 Copilot、改用原生函數」並強制人工複核（來源 6）。故落地分數略高於技術分。
  • 綜合判定 3：屬「可用但需人工把關、已有成熟產品」的 3-4 帶下緣；考量描述統計這一最常見子任務已可靠商品化，定 3。

實證一：Benchmark（測項＋分數＋日期＋來源）

  • InfiAgent-DABench（257 個資料分析挑戰、52 個 CSV、7 類別）：GPT-4o 無 agent 框架達 75.9%；原論文 GPT-4 78.99%，最佳開源 Qwen-72B-Chat 59.92%，落後 GPT-4 約 19 個百分點（2024-01，來源 2、4）。
  • DABStep（450+ 真實財務分析多步任務，含跨文件推理）：簡單題人類做 3 小時以上約 62%；難題最佳推理 agent o3-mini 僅 16%、R1 13%、Claude 3.5 Sonnet 12%、DeepSeek V3 6%（2025-02，來源 1）。
  • 解讀：描述統計／單步繪圖＝高分區（70-90%+）；多步真實分析＝低分區（<20%），是當前主要技術瓶頸。

實證二：真實部署/產品案例（誰用、規模、成效）

  • OpenAI ChatGPT Advanced Data Analysis（前身 Code Interpreter）：GA 功能，每次上傳上限 512MB，以實際 Python 運算（來源 4 對照、MIT Sloan 教學採用）。
  • Microsoft Excel Copilot：「Copilot in Excel with Python」已 GA；=COPILOT() 函數 2025-08 推 Insider、預計 2026-12 桌面/網頁版全面 GA（來源 3、5）。內建於 Excel 計算引擎，自然語言→自動生成並插入 Python 代碼出圖表/樞紐分析。
  • Anthropic Claude：企業客戶 2025-08 突破 30 萬家；32% 企業 LLM 工作負載跑在 Claude 上；金融業 24% 大型銀行採用（來源:Julius/IntuitionLabs 彙整）。單檔上限 20MB。
  • Julius AI 等垂直工具：處理上限達 32GB（高 RAM 容器），補足通用助理檔案大小限制（來源:Julius 比較頁）。
  • 規模成效：屬數百萬級日常使用的成熟功能；但企業高風險場景普遍要求人工複核，少有「無人把關自動產出對外結論」之規模化部署。

現況與邊界（AI 目前做得到/做不到）

  • 做得到：乾淨單表的描述統計、相關性、分群、標準圖表、自然語言→可執行代碼、口語化一句結論。
  • 做不到/不可靠：多步跨表/跨文件分析、需正確業務口徑的真實分析（DABStep 難題 <20%）；結果可重現性差（同輸入不同輸出）；會產生看似合理的錯誤公式/口徑；高風險財務、統計建模微軟建議「跳過、人工複核」（來源 6）。
  • 邊界一句話：「快出初步洞察」已可靠商品化；「可直接信任的最終分析結論」仍需人類把關。
