PDF文件OCR結構化

一句話定義（這個可交付物是什麼）

給定印刷體 PDF 或掃描影像，模型自動辨識文字並還原版面與表格結構，輸出可程式化使用的 Markdown / JSON / HTML（含標題層級、段落、表格 cell 結構、公式 LaTeX、閱讀順序）。屬「文件智能（Document Intelligence / IDP）」的核心環節。

難度評分（綜合 3/10：技術 2、落地 4）— 理由

  • 技術成熟度 2（成熟）：印刷體與一般版面的文字辨識已近飽和。OmniDocBench（CVPR 2025）上前段模型綜合分數已達 94-96 分，GLM-OCR 94.6 分超越 Gemini 3 Pro 與 GPT-5.2 等前沿模型（LlamaIndex 2026-02 直接以「benchmark 已飽和」為題）。GPT-4o 純字元編輯距離低至 0.020，文字準確度極高。技術已是商品。
  • 落地難度 4（可用但需人工把關）：核心難點不在技術而在可靠性長尾與信任。複雜表格（合併儲存格、多層表頭、多表並置）、手寫、財報/法律/保險表單仍會失敗；通用多模態模型在結構敏感指標（表格 TEDS、閱讀順序）上仍輸專用 OCR；高風險場景（醫療診斷、法律判斷、金融決策）廠商明文「不建議直接使用」。多數企業仍需 human-in-the-loop 抽檢。
  • 綜合判定 3：可用、已大規模商品化、但高價值/高風險文件仍需人工複核 → 落在量表 3-4「可用但需人工把關、已有產品」。較 P1 初判 3 維持不變。

實證一：Benchmark（測項＋分數＋日期＋來源）

  • OmniDocBench（綜合 ≈ ((1−文字編輯距離)×100 + 表格TEDS + 公式CDM)/3，越高越好），2026-05-21 榜（v1.5/v1.6 混榜，跨列非完全可比）：PaddleOCR-VL-1.6 = 96.33、MinerU2.5-Pro = 95.69（皆 v1.6）；GLM-OCR = 94.62、Gemini 3 Pro = 90.33、GPT-5.2 = 85.4、GPT-4o = 75.02（皆 v1.5；閉源模型僅供參考、不列入最佳排名）。[2][1]
  • 純字元準確度：CodeSOTA 榜列 GPT-4o 文字編輯距離 0.020；Mistral OCR-3 0.099、閱讀順序 91.63。[2]（注意：GPT-4o 綜合僅 75.02，OmniDocBench v1.5 論文列其全頁編輯距離約 0.217，0.020 疑為不同指標變體，勿據此判定 GPT-4o 字準最佳。）
  • 表格結構 TEDS：GLM-OCR 86.0 vs Gemini 3 Pro 81.8（GLM 勝）；但 Gemini 3 Pro 在手寫 KIE 94.5 反勝 GLM 86.1 → 手寫仍是弱項。[4]
  • 其他：Mistral OCR-4 OlmOCRBench 85.20、複雜文件 OmniDocBench 93.07；宣稱 25+ 語言 99%+ 字準。[3]
  • 注意：LlamaIndex（2026-02-24）指 OmniDocBench 已飽和、資料型別偏少偏易；其 TEDS／編輯距離會懲罰「語意正確但格式不同」的輸出（文中例：TEDS 0.6344、0.3825，及 LlamaParse 0.3825/編輯距離 0.5998），故榜分未必反映真實長尾失敗。[4]

實證二：真實部署/產品案例（誰用、規模、成效）

  • 三大雲商規模化部署：Azure Document Intelligence（並入 Foundry / Form Recognizer、可容器化 on-prem）、AWS Textract（S3+Lambda 大量 ingestion）、Google Document AI（Vertex AI、多語 OCR），均為企業生產級服務。導入後業界報告處理準確度 +30%、人工輸入工作量 −40%。[5]
  • 市場規模：Document AI 市場 2025 估 146.6 億美元 → 2030 預估 276.2 億美元，CAGR 13.5%（MarketsandMarkets）。已是成熟商業市場，非實驗室階段。[5]
  • 專用 OCR 產品：Mistral OCR 標準 API $4/千頁、批次 $2/千頁、吞吐達 2,000 頁/分鐘；客戶 Rogo（金融）用於圖表密集財報，「準確度與頂級 agentic 解析器持平、成本約 1/8、延遲約 1/17」。[3]
  • 開源端：MinerU、PaddleOCR-VL、dots.ocr 等可自架，降低落地門檻。

現況與邊界（AI 目前做得到/做不到）

  • 做得到：印刷體、標準版面、一般表格 → 高準確度結構化輸出（Markdown/JSON/HTML+LaTeX），已商品化、低成本、可自架。
  • 仍吃力/需人工：合併儲存格與多層表頭的複雜表格、手寫、財報/法律/保險等高密度表單、視覺圖表元素、跨欄閱讀順序；LLM 式解析偶有幻覺（補字/改數）。
  • 不宜直接信任：醫療、法律、金融等高風險決策文件，廠商明示不適用，須人工複核。本任務的瓶頸已從「能不能辨識」轉為「可靠性長尾與信任邊界」。
