客製AI晶片ASIC

一句話定義

客製 AI 晶片 ASIC（Application-Specific Integrated Circuit，特殊應用積體電路）是超大規模雲端業者（hyperscaler）為自家 AI 工作負載量身打造的專用加速器，犧牲通用 GPU 的彈性，換取在特定模型架構（多為 Transformer 推論）上更高的每瓦效能與更低的每 token 成本，代表產品包括 Google TPU、AWS Trainium / Inferentia、Meta MTIA、Microsoft Maia；其戰略目的是降低對 NVIDIA GPU 的依賴。

關鍵事實與數據（每項附來源與時間）

  • ASIC 與 GPU 的核心差異：ASIC 針對單一類任務（通常是某家公司的特定模型架構或推論模式）做到極致效率，提供更高效能/瓦與更低每 token 成本，但在該用途之外幾乎沒有彈性（Tom’s Hardware, 2026-05；Tech Times, 2026-05）。
  • 起源：Google 約 2014–2015 年於資料中心部署第一代 TPU，從專案啟動到上線僅 15 個月，技術負責人為 Norman Jouppi；2016 年 Google I/O 公開（Wikipedia / The Chip Letter，引 2017 ISCA 論文）。
  • 代表論文：Jouppi 等人 2017 年 ISCA 論文〈In-Datacenter Performance Analysis of a Tensor Processing Unit〉（arXiv:1704.04760）指出第一代 TPU 較當時 CPU/GPU 有 15–30 倍效能、30–80 倍效能/瓦；核心為 65,536 個 8-bit MAC 的脈動陣列（systolic array），峰值 92 TOPS（arXiv, 2017-04）。
  • Google TPU v7「Ironwood / TPU7x」單晶片：FP8 峰值 4,614 TFLOPs、BF16 2,307 TFLOPs；HBM 192 GiB、頻寬 7,380 GiBps；單晶片由兩顆 chiplet 組成（Google Cloud 官方文件，2025）。
  • Ironwood 整櫃（superpod）：9,216 顆晶片，FP8 約 42.5 exaFLOPs、BF16 約 21.3 exaFLOPs，合計 HBM 約 1.77 PB；採 3D torus 互連（Google Cloud 官方文件 / TrendForce, 2025-11）。CNBC 引述單晶片 FP8 4.6 petaFLOPS，略高於 NVIDIA B200 的 4.5 petaFLOPS，惟須注意 Ironwood 為 FP8、B200 此數常以 FP16 計，口徑不同（CNBC, 2025-11-21；The Register, 2025-11-06）。
  • AWS Trainium3：2025 年 12 月發表，AWS 首顆 3nm AI 晶片，單晶片 FP8 約 2.52 petaFLOPs、HBM3e 144 GB、頻寬 4.9 TB/s（CNBC, 2025-11-21）。
  • 成本優勢（業者數據，宜謹慎看待）：Ironwood 單晶片 TCO 較 GB200 server 約低 44%（Tom’s Hardware, 2026-05）；Midjourney 自述由 NVIDIA GPU 遷移至 Google TPU 後月運算成本由 210 萬美元降至 70 萬美元（約 -65%，業者單一案例，資料未經獨立驗證）。
  • 委外設計集中度：Broadcom 與 Marvell 合計約掌握 95% 客製 AI ASIC 共同設計（co-design）市場（Tom’s Hardware, 2026-05）。

在 AI 供應鏈的位置與角色

  • 屬上游「運算晶片」環節，是 NVIDIA / AMD 商用 GPU 的替代或互補方案；買方即設計方（Google、Amazon、Meta、Microsoft、OpenAI 等超大規模業者）。
  • 價值鏈分工：hyperscaler 出規格與模型需求 → Broadcom / Marvell 做 IP、SerDes、整合與後段設計 → 台積電 代工先進製程（3nm/5nm）並做 先進封裝CoWoS → 搭配 HBM記憶體（HBM3e）。
  • 應用偏好：ASIC 主要切入「推論（inference）」市場——工作負載可預測、量體大到足以攤平設計投資；NVIDIA 仍主宰「訓練（training）」，因 CUDA 彈性與 NVLink 多晶片擴展難以複製（Tech Times / Built In, 2026）。
  • 與 Transformer 強綁定：多數 ASIC 的硬體加速單元（如 attention、FFN）直接針對 Transformer 推論模式設計，因此其競爭力與主流模型架構是否續用 Transformer 高度相關。

現況與最新動態（2024–2026）

  • 市占格局：NVIDIA 約仍握 AI 晶片市場 70%（Tom’s Hardware, 2026-05）；但 2026 年客製 ASIC 出貨量預估年增 44.6%，近乎商用 GPU 16.1% 增速的三倍，ASIC 伺服器出貨佔比上看 27.8%（Tom’s Hardware / Tech Times, 2026-05）。
  • AWS「Project Rainier」：2025 年為 Anthropic 啟用、含近 50 萬顆 Trainium2 的叢集，計畫 2025 年底突破百萬顆；提供 Anthropic 逾 5 倍於前代訓練的算力（AWS / DCD, 2025）。
  • Google：2025 年 11 月推出第七代 Ironwood TPU，主打可同台抗衡 NVIDIA Blackwell/GB300，並傳出與 Meta 洽談數十億美元 TPU 供應案（Tom’s Hardware, 2026-05）。
  • Meta MTIA：採約半年一代的快速迭代節奏，路線圖含 MTIA 300/400/450/500，由純推薦/排序推論擴展至 GenAI 推論；MTIA 450 規劃 FP8 7 petaFLOPs、HBM 288 GB、TDP 1,400W，預計 2027 年初量產（Meta AI Blog / DCD, 2025–2026）。
  • Broadcom：FY2026 Q1 AI 營收 84 億美元（年增 106%），AI 在手訂單 730 億美元；OpenAI 透過 Broadcom 規劃 10GW 客製加速器，預計 2026 下半年部署（Tom’s Hardware, 2026-05）。

危機 / 變化 / 風險

  • 軟體生態護城河：NVIDIA CUDA 具網路效應（每個新 CUDA 優化模型/函式庫都讓生態對下一位開發者更有價值），ASIC 須自建編譯器與框架支援，遷移成本高（PitchGrade / Built In, 2025–2026）。
  • 「加層而非取代」：截至目前無任何 hyperscaler 實質削減 NVIDIA 採購，客製矽多是增量補強而非替換（Tech Times, 2026-05）——意味 ASIC 成長未必等於 NVIDIA 衰退。
  • 彈性風險：ASIC 為特定架構最佳化，若主流模型架構轉向（如後 Transformer 架構），既有設計可能快速貶值；流片（tape-out）成本高、開發週期長，押錯方向代價巨大。
  • 供應鏈集中：設計高度仰賴 Broadcom/Marvell（合計約 95%）、製造仰賴台積電先進製程與 CoWoS、記憶體仰賴 HBM，任一環節瓶頸都會限制產能。
  • 數據可信度：多數 TCO / 省錢數字來自業者或單一客戶自述（如 Midjourney、Google 對比 GB200），缺乏獨立第三方統一基準，跨晶片比較常因精度（FP8/BF16）、系統層級（晶片 vs 整機）定義不同而失真，引用時須標明口徑。
