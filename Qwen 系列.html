Qwen 系列

一句話定義

Qwen（通義千問）是阿里巴巴／阿里雲開發的大型語言模型家族，以「開源權重 + 多尺寸 + 多模態」策略成為全球下載量最高的開源模型生態系，並以閉源旗艦（Qwen-Max 系列）對標 GPT/Gemini/Claude 頂尖閉源模型。

關鍵事實與數據（每項附來源與時間）

  • 開發者：阿里巴巴集團 / 阿里雲，由 Qwen 團隊負責（Wikipedia，2026）。
  • 起源時間線：初代 Qwen 2023-04 beta、2023-09 公開；Qwen2 2024-06；Qwen2.5 2024-09；Qwen3 2025-04-28/29；Qwen3-Max 2025-09-23；Qwen3.5 2026-02-16（Wikipedia / Alibaba Cloud blog / Artificial Analysis）。
  • Qwen3 旗艦（開源）：Qwen3-235B-A22B，MoE 架構，總參數 235B、每 token 啟動 22B；採 Apache 2.0 授權；2025-04-29 發布（官方 Qwen blog / Alibaba Cloud）。
  • Qwen3 完整陣容：6 個密集模型（0.6B / 1.7B / 4B / 8B / 14B / 32B）＋ 2 個 MoE（30B-A3B、235B-A22B），全數 Apache 2.0；密集模型 context 32K 或 128K、MoE 128K（官方 Qwen blog，2025）。
  • Qwen3 代表 benchmark（Qwen3-235B-A22B-2507，官方 2025-07）：MMLU-Pro 84.4、MMLU-Redux 93.8、GPQA 81.1、AIME2025 92.3、LiveCodeBench 74.1、Arena-Hard 1422、IFEval 87.8（Qwen 官方數據，經多源轉述）。

  • Qwen3-Max（閉源旗艦）：2025-09-23 發布，逾 1 兆參數，context 約 260K、輸出上限約 66K；Artificial Analysis 智慧指數 31（71 個模型中排約第 24，平均 24）；定價約 US$1.2/M 輸入、US$6.0/M 輸出（Artificial Analysis / eesel，2026）。
  • Qwen3.5-397B-A17B（現主力旗艦）：2026-02-16 發布，MoE 397B 總參數、僅啟動 17B、512 experts；開源版 256K context、託管版 Qwen3.5-Plus 達 1M token；支援 201 種語言；Artificial Analysis 智慧指數 45，於同級開源模型中名列前茅，並宣稱在多項推理／coding 任務上勝過自家逾兆參數的 Qwen3-Max（VentureBeat / MarkTechPost / Artificial Analysis，2026-02）。

在 AI 供應鏈的位置與角色

  • 中游（模型層）核心玩家：Qwen 提供從 0.6B 邊緣裝置到逾兆參數雲端旗艦的完整尺寸帶，同時橫跨文字、視覺、音訊、全模態與影像生成。
  • 開源生態地基：因 Apache 2.0 授權與龐大衍生模型數，Qwen 已成為全球（尤其中文／亞太）開發者微調與商用的預設底座，類似 Linux 在作業系統的地位；上游依賴 NVIDIA GPU 訓練，下游被 Hugging Face 託管並由大量新創與企業二次開發。
  • 雙軌商業模式：開源權重衝生態與品牌、閉源 Qwen-Max 系列透過阿里雲 Model Studio 變現，與 OpenAI、Google、Anthropic 的純閉源路線形成對比，更接近 Meta Llama 與 DeepSeek 的開放路線。

危機 / 變化 / 風險

  • 地緣政治與信任風險：作為中國模型，在歐美政府／金融等敏感場景面臨資料安全疑慮與潛在出口管制／採購限制；國際企業導入時常受合規審查。
  • 算力與制裁風險：訓練高度依賴 NVIDIA 先進 GPU，美國對中出口管制可能限制下一代旗艦的擴張節奏。
  • 開源變現張力：開源衝高下載量但直接 API 營收有限，須靠阿里雲生態回收，商業模式可持續性受 DeepSeek、Llama 系列 等開源同業價格戰擠壓。
  • 版本碎片化與資訊雜訊：發布節奏極快（Qwen3 → 3.5 → 3.6/3.7 在一年內），第三方榜單與社群轉述常領先官方文件，造成版本與數據混淆，引用時須回到官方模型卡／技術報告核實。
  • benchmark 可比性：官方數據多為自家測試條件，跨模型比較須以 LMArena／Artificial Analysis 等第三方獨立評測交叉驗證。
