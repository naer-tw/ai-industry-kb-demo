Anthropic

一句話定義

Anthropic 是由前 OpenAI 高層創立的前沿 AI 實驗室，以「安全導向」為核心定位，開發 Claude 系列大型語言模型；在 AI 供應鏈屬中游「模型／基礎模型層」，2025–2026 年靠企業 API 與 Claude Code 爆發式成長，已超越 OpenAI 成為企業級 LLM 市占龍頭。

在 AI 供應鏈的位置與角色

Anthropic 位居 AI 供應鏈的中游「基礎模型／模型層」，角色是把上游算力轉化為可商用的智慧 API：

  • 對上游（算力供應）：採多元晶片策略，同時使用 AWS Trainium、Google TPU 與 NVIDIA GPU。與 Amazon 共建 Project Rainier，使用逾百萬顆 Trainium2 晶片訓練／服務 Claude（來源 4）；與 Google 擴大合作，規劃用至多 100 萬顆 TPU、2026 年帶來逾 1GW 算力（來源 5）。
  • 對下游（應用端）：透過 Claude API、AWS Bedrock、Google Cloud、Microsoft Foundry 對外供模型；逾 30 萬企業客戶在其上建構應用，10 萬家以上於 Amazon Bedrock 上跑 Claude（2026-04）。Claude Code 直接切入軟體開發市場。
  • 與雲端業者的雙重關係：Amazon／Google 既是 Anthropic 的最大投資者與算力供應商，也透過持股共享其估值增長（Fortune 2026-04 報導，兩大雲端「亮眼 AI 獲利」中約半數來自 Anthropic 持股而非本業）。
  • 定位差異：相對 OpenAI 走全市場（含消費端 ChatGPT），Anthropic 聚焦企業、coding 與「安全可信賴」賣點。

現況與最新動態（2024–2026）

  • 模型迭代加速：Claude Opus 4.5（2025-11-24）為首個 SWE-bench Verified 突破 80% 的模型（80.9%），超越 GPT-5.1（76.3%）與 Gemini 3 Pro（76.2%），主打「全球最佳 coding／agent／computer use 模型」（來源 6、Vellum）。
  • 最新旗艦 Claude Opus 4.8（2026-05-28）：Anthropic 目前最強通用模型，新增 Claude Code「dynamic workflows」處理大規模任務、可調節 effort 等級，Fast 模式可達 2.5 倍速且較前代便宜三倍；於 claude.ai 與 AWS／Google Cloud／Microsoft Foundry 上線（來源 7、TechCrunch 2026-05-28）。
  • 算力大擴張：2026-04 Amazon 加碼，承諾未來十年投入逾 1,000 億美元於 AWS、取得至多 5GW 算力（來源 4）；2025-10 與 Google 簽署「數百億美元」TPU 雲端協議（來源 5）。
  • 企業反超 OpenAI：2025 年起多份報告顯示 Anthropic 於企業 LLM 用量／支出市占登頂，且訓練成本據稱僅 OpenAI 約 1/4（SaaStr 報導，成本數字未經官方證實）。

危機 / 變化 / 風險

  • 安全承諾鬆動：2026-02 Anthropic 修改沿用兩年的「負責任擴展政策（RSP）」，改採可隨時變動的非綁定式安全框架，被批評在與五角大廈的 AI 紅線爭議中放棄核心安全承諾（來源 10，CNN 2026-02-25）；CEO Dario Amodei 亦坦言公司在「安全 vs. 商業壓力」間掙扎（Fortune 2026-02-17）。此與其創立初衷的「安全替代方案」形象產生張力。
  • 版權訴訟：2025-09 與作家群體達成首例 AI 版權集體訴訟和解，支付 15 億美元（約 50 萬本書、每本約 3,000 美元），因被控以盜版書籍訓練 Claude（來源 9，NPR）；UMG／Concord 的 31 億美元音樂版權訴訟仍進入證據開示階段。
  • 算力與資本依賴：對 Amazon／Google 的投資與雲端算力高度依賴，形成「投資者即供應商」的綁定關係，議價與戰略自主性受限。
  • 燒錢與獲利路徑：估值近 1 兆美元、run-rate 暴增，但前沿模型訓練與推論成本巨大，獲利能力與「AI 泡沫」疑慮為市場關注焦點。
  • 消費端落後：消費市場遠不及 ChatGPT，營收高度集中於企業／API，客戶集中度與單一賽道風險偏高。
