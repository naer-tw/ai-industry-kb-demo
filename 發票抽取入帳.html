發票抽取入帳

一句話定義（這個職業任務的可交付物是什麼）

給定一張採購發票影像（掃描檔／PDF／照片），系統自動辨識並結構化抽取關鍵欄位——供應商、發票號碼、日期、未稅金額、稅額、總額、品項明細（line items）——再與採購單（PO）／驗收單（GRN）三方匹配、產生 GL 會計科目分類與「自動入帳建議」，於可信度達標時無須人工碰觸即完成過帳（touchless / 直通處理 STP）。屬應付帳款自動化（AP Automation / IDP）的核心職業任務，取代會計事務人員的鍵入與初核。

難度評分（綜合 4/10：技術 3、落地 5）— 理由

  • 技術成熟度 3（可用、已商品化但需把關）：發票欄位抽取已是成熟商品。LlamaExtract 在 2025-10 一份評測中整體欄位準確率 94%（基準 Docling 僅 63%），標準化欄位（發票表頭、bill-to）最高，行項目較難為 91%，一致性檢查通過率 93%、數學驗證錯誤 0。[1] 多家前沿多模態模型（Claude Sonnet 3.5、Gemini 2.0 Pro、GPT-4o）在公開發票上皆可達高準確抽取，Claude Sonnet 3.5 在跨畫質測試整體準確率與韌性最佳（含低畫質影像）。[2][3] 技術瓶頸集中在長尾：行項目多語多格式、合併欄位、手寫、皺摺/歪斜照片、稅務拆分（多稅率、含稅/未稅）。
  • 落地難度 5（前沿可做但可靠性/責任/流程是真正障礙）：難點不在「看懂發票」，而在「敢自動入帳」。STP 85%+ 並非單純抽取精度，而是端到端 touchless——需 PO/GRN 三方匹配通過、科目分類正確、核准規則無例外，缺一即落入人工 exception。財務責任、稽核軌跡、SOX/內控、稅務合規（如台灣統一發票字軌、進項稅額憑證）、供應商主檔比對、重複付款防呆等，都壓低可自動過帳的比例。產業實況：best-in-class 平均 touchless 率僅 49.2%、業界平均 32.6%（Ardent Partners 2025），85%+ 屬「PO-backed、高量、成熟部署」才達得到的上緣。[6]
  • 綜合判定 4：技術可做、產品成熟、低風險場景已落地，但「85%+ STP」需條件成熟（PO 紀律＋信心門檻＋例外處理）且仍受監管與財務責任約束 → 落在量表 3-4 之間偏 4。較 P1 初判 3 略升，反映落地長尾與責任門檻。

實證一：Benchmark（測項＋分數＋日期＋來源）

  • 欄位級整體準確率：LlamaExtract 整體 94%，行項目（line items）91%，表頭與 bill-to 最高（標準化欄位）；對照基準 Docling 僅 63%（行項目 58%、稅/淨額/幣別 60%）。[1]（2025-10）
  • 跨模型／跨畫質：20 張公開發票、400+ key-value pairs，準確率＝正確 KV 數 / 總 KV 數；Claude Sonnet 3.5 整體準確率與韌性最佳，低畫質影像優勢明顯。測試完成 2024-12（含 Amazon Textract、Google Document AI、Azure Document Intelligence）。[2]
  • 多工具對比：7 套方案（AWS／Azure／Google Document AI／GPT-4o 文字與影像／Gemini 2.0 Pro／Deepseek v3）、20 張發票（2006–2020）。Gemini 2.0 Pro 細節抽取最佳（樣本可達文字與數值 100%）；GPT-4o(影像) 數值偶有誤；Azure 在多詞品項描述上最差（單張效率僅 33.3%）；Google Document AI 未能輸出結構化品項（合併為單一字串）。[3]（2025-03）
  • 限制：上述多為小樣本（≈20 張）、欄位混合計分，缺大規模分欄位 F1，故 evidence 列 B 而非 A。稅額與行項目通常是準確率最低的兩欄。

實證二：真實部署/產品案例（誰用、規模、成效、監管限制）

  • Vic.ai：六個月內客戶可達 up to 85% no-touch 發票率、70% Autopilot；某運輸業客戶達 99% 編碼準確率、84% no-touch，並釋出 3 名 FTE 轉高價值工作；一份試點四週學習期後 ≈78% 發票免人工過帳。[4]
  • Basware：客戶 PO-backed 發票 touchless 率 60–85%，成熟高量部署 90%+；其自身內部部署 2025-03 達 77% touchless match。[5]
  • 產業基準（Ardent Partners 2025）：best-in-class touchless 49.2%、業界均 32.6%；最佳發票例外率 9%（業界均 22%）；最佳每張成本 $2.78（業界 $9.40、重人工 $12.88）；最佳處理 3.1 天（其他 17.4 天）。顯示 85%+ 是上緣而非常態。[6]
  • 門檻實務：KlearStack 建議 STP 門檻設 85%+ 取得自動化與準確的最佳平衡。[7] 監管/責任限制：高金額、無 PO、稅務存疑、供應商首次往來等，業界普遍強制人工覆核，不放行自動過帳。

現況與邊界（AI 目前做得到/做不到）

  • 做得到：標準採購發票（尤其 PO-backed、固定供應商格式）的供應商／金額／稅額／品項抽取與科目建議；在信心門檻控管下，成熟部署可達 85%+ touchless。屬可大規模商品化的職業任務。
  • 做不到／需人工：(1) 端到端「全自動入帳」的全面無人化——例外（無 PO、三方不匹配、稅額拆分異常、重複付款疑慮、新供應商）仍須人工；(2) 跨稅制/多國/手寫/劣質影像長尾；(3) 財務責任、稽核與內控要求下的高風險過帳（高金額、合規敏感）；(4) 將「抽取準確率 94%」直接等同「STP 94%」——兩者不同，STP 受流程紀律拖累更低。85%+ STP 是「條件達成」而非「開箱即得」。
