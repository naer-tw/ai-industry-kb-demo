AI產業知識庫capability

多表商業資料分析

一句話定義（這個可交付物是什麼）

給一份由多張表（CSV/JSON/資料庫）＋非結構化說明文件（Markdown 手冊、欄位字典）組成的「異質」商業資料集，加上一句「含歧義、需多步推理」的分析需求（例：「找出去年高價值客戶為何流失」），系統要自己釐清需求、跨表 JOIN、套用領域規則、做中間計算，最後輸出端到端正確的分析結論並附佐證（數字、推理鏈、查得到的依據）。難點在「歧義 + 多步 + 端到端正確」三者同時成立，而非單純把一句話翻成一條 SQL。

難度評分（綜合 6/10：技術 6、落地 7）— 理由

  • 技術 6（前沿可做、可靠性仍是瓶頸）：單表查詢、text-to-SQL 已成熟（Snowflake Cortex Analyst 內部 BI benchmark 約九成準），InfiAgent-DABench（單表為主）GPT-4 約 75–79%。但一旦進入「多表異質＋歧義＋多步」的真實情境，benchmark 急遽掉：DABStep Hard 題基線最佳僅 14.55%（o4-mini），即使 2025/11 最先進 agent（Google DS-STAR）也只把整體拉到 45.2%。代表「端到端正確」這件事目前過半失敗。
  • 落地 7（歧義＋資料品質＋信任是主障礙）：(1) 歧義是頭號殺手——業界 CDAO 指出「沒有統一語意脈絡時，agent 只會回答字面解讀，而那很少是對的問題」；(2) 逾九成 AI 失敗源於資料品質（欄位定義不一、髒資料、跨倉分散），非模型本身；(3) 「200 OK / 95ms」可以是完全錯的答案，需可觀測性與人工複核，高風險決策不敢全自動。故落地分高於技術分。
  • 綜合判定 6：屬量表 5–6 級「前沿可做、部分落地，但可靠性/成本仍挑戰」的上緣，因 Hard 題端到端正確率過半失敗、需強人工把關而不取 5。
  • 查證重評：6/6/7（vs P1 初判 6，差 0；微調 diff_tech 由隱含 6 維持、diff_deploy 上調至 7）——獨立比對錨定量表：與既有 企業知識庫帶引用問答（5/4/6）相比，本任務多了「跨多表 JOIN＋多步計算＋歧義消解」的複合要求，技術與落地皆更難一級；但商用平台（Snowflake/Databricks）已規模化部署語意層+agent，故不到 7–8（實驗室級）。三分與 P1 一致或上調 1，結論一致，confidence: high。（2026-06-26 查證）

實證一：Benchmark（測項＋分數＋日期＋來源）

  • DABStep（Adyen × Hugging Face，2025/06）：450+ 道源自金融營運的真實任務，須結合結構化表（CSV/JSON）＋非結構化說明文件（Markdown 手冊）做多步推理，約 84% 為 Hard。Hard 題明確「抗單發程式生成」，需迭代過濾、跨表交叉引用、套領域規則。基線：Easy 約 76%（o4-mini 76.4、GPT-4.1 80.6、Claude 3.5 Sonnet 77.8）；Hard 最佳僅 14.55%（o4-mini），Claude 3.7 Sonnet 13.76、o3-mini 13.76、Gemini 2.5 Pro 12.70、GPT-4.1 12.43。Easy↔︎Hard 落差約 62 個百分點。論文無人類基線。[來源1][來源2]
  • DS-STAR（Google，2025/11/06）：以「資料檔分析模組（自動抽取各式格式含非結構化的脈絡）＋序列式規劃（規劃-寫碼-驗證迭代最多 10 輪）」架構，登上 DABStep 公開榜首，整體準確率由前最佳 41.0% 提升至 45.2%。以 Gemini-2.5-Pro 與 GPT-5 為底模測試：GPT-5 在 Easy 較佳、Gemini-2.5-Pro 在 Hard 較佳。判讀：頂尖 agent 框架仍有過半題目做不到端到端正確。[來源3]
  • InfiAgent-DABench（2024，首個資料分析 agent benchmark）：603 題、124 個 CSV（以單表/受控題為主）。GPT-4 約 74.6–79%，最佳開源（Qwen-72B-Chat）落後約 17%。對照組顯示「單表、題意清楚」時準確率明顯高於 DABStep 的多表歧義情境。[來源4]

實證二：真實部署/產品案例（誰用、規模、成效）

  • Snowflake Cortex Analyst / Snowflake Intelligence（2025/11 GA）：企業級自然語言分析 agent。官方工程部落格稱在「代表真實 BI 風格問題」的內部 benchmark（150 題涵蓋銷售/行銷/財務）上 text-to-SQL 準確率 ≈90%+，較 GPT-4o 單發生成（同 benchmark 僅 51%）約 2 倍、較市面其他方案約高 14 個百分點，靠語意模型（semantic model：使用者定義度量/篩選一致套用）+ agentic 系統 + 多 gold query 彈性計分達成。針對星狀/雪花綱要的多表 JOIN，另以 join-path 驗證緩解 JOIN 幻覺、fan/chasm trap 與重複計算，該情境約 92% 準確（vs 市面 ~73%、GPT-4o 55%）。屬已規模化商用，但成效高度依賴事前定義好的語意模型（semantic view）。[來源5][來源7]
  • Databricks（2025–2026）：以 ai_parse 等 SQL 原生方式把 OCR/RAG/抽取整併為單一宣告式語句，降低串接 LLM 的幻覺風險；定位為治理優先的 agentic 分析。屬規模化平台能力。[來源6 衍生]
  • 市場採用：Gartner 預測至 2026 年底約 40% 企業導入 AI 增強分析。但業界普遍回報失敗模式集中於：歧義需求被字面解讀、跨倉資料分散缺統一脈絡、>90% 失敗源於資料品質——故多數生產部署仍鎖在「人工把關 + 受控語意層」內，而非開放式端到端自動分析。[來源5 衍生]

現況與邊界（AI 目前做得到/做不到）

  • 做得到：題意清楚、語意模型/欄位字典已備、單表或少量表時，可生成高準確（約九成）查詢與分析，並附 SQL/推理作佐證；已有規模化商用產品（Snowflake、Databricks）。
  • 做不到/不穩：(1) 「多表異質＋歧義＋多步」真實情境下端到端正確率仍 <50%（DABStep Hard 即使最強 agent 45.2%）；(2) 歧義消解差——常回答字面而非真正該問的問題；(3) 對髒資料、跨倉一致性、領域規則套用脆弱；(4) 自動驗證/可觀測性不足以單獨守門，高風險決策須人工複核。整體屬「前沿可做、部分落地，但端到端可靠性仍是主要挑戰」。
