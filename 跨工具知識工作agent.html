AI產業知識庫capability

跨工具知識工作agent

一句話定義（這個職業任務的可交付物是什麼）

給一個真實的多步驟知識工作任務——需跨工具完成「上網查資料 → 跨來源比對核實 → 產出結構化檔案（報告/試算表/簡報）」——讓 AI agent 在「同一任務獨立重跑 8 次」的嚴格條件下，至少 8 次中 80% 以上各自獨立成功（pass@8 ≥80%，本契約取「每次重跑皆視為一次獨立成功率」的穩定性口徑，逼近 τ-bench 的 pass^k 一致性精神）。重點不在「能不能做一次」，而在「能不能可重複、可信賴地一直做對」。

難度評分（綜合 8/10：技術 7、落地 8）— 理由

  • 技術 7（前沿可做單次、但穩定性是未解瓶頸）：跨工具多步驟任務的「單次能力」已到前沿可用——GAIA 上 Claude Sonnet 4.5 總分 74.6%（2026Q2）、最難的 Level 3 也達 65.4%（2025/09）。但本任務的硬條件是「重跑下 ≥80% 穩定」，而 agent 的 run-to-run 變異極大：τ-bench 的 pass^k（k 次全對的機率）顯示 GPT-4o 在 retail 的 pass^8 <25%；理論上一個 pass@1 90% 的模型，pass^8 仍只剩約 57%。要在多步驟（每步都可能掉鏈）任務上達到 pass@8 ≥80%，等同要求單步可靠度近乎完美，目前模型做不到。故技術分高於「可用需把關」的 5–6 區，落在 7。
  • 落地 8（可靠性即信任，職業場景門檻最高）：知識工作的產出要被當成「可交付」，錯一次就可能誤導決策。實務調查指出可靠性是 agent 落地最大障礙，企業因此「放棄開放式/長時程任務，退回少步驟＋人工複核的工作流」；實驗室 benchmark 與真實部署存在約 37% 落差、相近準確率成本可差 50 倍。職業任務還疊加責任歸屬、稽核軌跡、權限隔離，落地分數高於技術，給 8。
  • 綜合判定 8：屬「實驗室/早期可做、可靠性低或落地障礙大」的 7–8 區。P1 初判 7 上修為 8——關鍵在契約的「pass@8 ≥80% 穩定」門檻把任務從「能做一次」拉到「可重複信賴」，這是目前 agent 最弱的一環。

實證一：Benchmark（測項＋分數＋日期＋來源）

  • GAIA（General AI Assistants，arXiv 2311.12983；466 題，Level 1–3，測 web browsing＋檔案處理＋多步驟工具使用）：HAL（Princeton）排行榜上 Claude Sonnet 4.5 總分 74.55%（Level 1 82.1%、Level 2 72.7%、Level 3 僅 65.4%，2025/09）。2026Q2 各方報導前沿 agent 約 75%，人類基線 92%。判讀：跨工具多步驟「單次成功率」約七成五，最難的真實多步任務（L3）約六成五——離 80% 已有距離，且這只是 pass@1。[來源1][來源2]
  • τ-bench / τ²-bench（Sierra，Tool-Agent-User 互動，提出 pass^k＝k 次全對機率作為可靠度指標）：核心發現——SOTA function-calling agent（GPT-4o）retail 任務 pass^8 <25%（Sierra 原文：重跑同一任務 8 次僅約 25% 成功，較 pass^1 約掉 60%）；且「一個 pass@1 90% 的模型，pass^8 掉到約 57%」。τ²-bench 上即便 Gemini 3 Pro（2025-11-18 發布）pass^1 約 85%（retail）/73%（airline），pass1→pass4 仍明顯下滑。這是本契約最關鍵的反證：pass@8 ≥80% 的穩定門檻，遠超現有模型在重跑下的一致性。[來源3][來源4][來源5]
    • 查證註（2026-06-26）：pass^8<25%（GPT-4o retail）與「pass@1 90%→pass^8≈57%」已由 Sierra 部落格與 τ-bench 論文獨立佐證。Gemini 3 Pro 在 τ²-bench 的 85%/73% 為單一數據點，至 2026-06 未能於公開 leaderboard（HAL、Artificial Analysis）獨立查得，標為待補來源；其方向與「可靠度落差」論點一致，不影響結論。
  • 可靠度科學（2025–2026 多篇）：單次 pass@1 常較 pass@k 高 15–25 分；「90% benchmark ≈ 70% 生產可靠度」。若 benchmark 不報 N 次重跑可靠度，通常掩蓋 10–30 分變異。代表本任務真正的瓶頸是「可重複性」而非「能力上限」。[來源4]

實證二：真實部署/產品案例（誰用、規模、成效、監管限制）

  • OpenAI Deep Research（企業版，2026/02 起）：可連 MCP/企業知識庫（Google Drive/SharePoint/GitHub 等），產出含內嵌引用、推理軌跡、可稽核中間發現的結構化報告，作為後端服務可程式化串接多代理工作流。企業高影響用例：資料分析與報告產出 60%、56% 規劃 agent 主導研究。屬「跨工具查資料→產檔案」已商用化的代表。限制：官方承認仍會偶發幻覺、難辨權威來源與傳聞，需人工把關。[來源6]
  • 企業落地現況（2025–2026 調查與分析）：306 位實務者調查指可靠性是 agent 落地最大障礙；企業普遍「退回少步驟＋強制人工複核」的受控工作流，不敢交付開放式長時程任務；benchmark 與真實部署約 37% 落差、相近準確率成本差達 50 倍。即「單次跑得動的 demo」多，「重跑都穩到可無人交付」的規模化部署尚未成熟。[來源4]
  • 規模化「pass@8 ≥80% 無人交付」的公開案例：未證實。目前部署皆以 human-in-the-loop（人工複核）保障可靠度，本契約定義的「獨立重跑高一致性、可免人工守門」尚無公開規模化證據。

現況與邊界（AI 目前做得到/做不到）

  • 做得到：單次跑通跨工具的「查資料→比對→產出帶引用報告/試算表」，前沿 agent GAIA 約 75%、Deep Research 等產品已商用；有推理軌跡與引用可供人審。
  • 做不到/不穩：(1) 重跑一致性差——pass^8 在受控 benchmark 上常 <25–57%，遠達不到契約 80%；(2) 多步驟誤差累積，每步小機率失敗在 8 次重跑下被放大；(3) 真實任務（GAIA L3）單次也僅約 65%；(4) 高風險知識工作仍須人工複核，尚無「免守門、重跑皆穩」的規模化部署。整體屬「單次前沿可做、但 pass@8 ≥80% 穩定門檻目前難達成」。
