Excel試算表分析

一句話定義（這個可交付物是什麼）

使用者給一份資料集（CSV/Excel）＋一個分析問題（如「算各區季度成長率並做趨勢圖」），AI 要產出一個可在 Excel 開啟的試算表檔：內含正確的儲存格公式（非寫死數值）、樞紐分析、圖表與格式。關鍵差異不在「算出答案」，而在「交付一個檔案層級正確、公式可追溯、圖表可編輯」的 .xlsx 成果。比 CSV資料分析（在對話框出圖出結論）更難——它要求對 Excel 原生物件（公式語法、圖表、版面）的精準操作，是 SpreadsheetBench 衡量的核心能力。

難度評分（綜合 5/10：技術 5、落地 6）— 理由

  • 技術 5（前沿可做、可靠性分歧大）：通用前沿模型在 SpreadsheetBench（912 題真實 Excel 問題）長期落後人類——2024 GPT-4o 僅 18.35%、原生 Copilot 約 20%，遠低於人類專家 71.33%（來源1）。但 2026 出現分水嶺：專用 Excel agent 大幅追上——Decide 在 Verified（400 題）達 82.5%、GPT-for-Excel 92.5%、金山 Qingqiu 83–89%，已逼近或超越人類專家（來源2-4）；Copilot Agent Mode 也升到 57.2%（來源2）。差別在「會迭代、會驗證、綁原生 Excel 動作」的 agentic 系統 vs 單發生成的裸模型（裸 Claude Files 42.9%、o3 23.3%，來源2）。但難尾仍硬：SpreadsheetBench-V2（財務建模/除錯/視覺化）最強 Claude Opus 4.6 僅 34.89%、除錯類更只 12%（來源2）。故技術定 5：專用工具已可用，泛用可靠性與難題仍是瓶頸。
  • 落地 6（已 GA、但須強人工把關）：Copilot in Excel 與 Python in Excel 均 GA，能自動生公式（XLOOKUP/SUMIF）、樞紐、推薦並產出圖表；Agent Mode 2025-12 起 GA、可自動建表/寫公式/出圖並「迭代到驗證通過」（來源6-7）。但高分多為廠商在 Verified 子集自報，且生成公式可能看似合理卻錯、結果可重現性差，財務/統計高風險場景仍須人工複核——落地分高於技術分。
  • 綜合判定 5：屬量表 5-6「前沿可做、部分落地、可靠性/成本仍挑戰」的下緣。與 P1 初判 5 一致：相較 CSV資料分析（3）多了檔案層級原生 Excel 物件操作而更難；但較 多表商業資料分析（6）的「歧義＋多表端到端」要求略低，且已有規模化 GA 產品，故不取 6。confidence: high。
  • 查證重評：5/5/5（技術/落地/綜合，2026-06-26）：對照錨定量表獨立重評——技術 4–5（專用 agent 已過人類基線 71.33%，但裸模型雜題 ~20–43%、V2 難尾除錯僅 12%，落於 5–6 下緣）、落地 5–6（GA 規模化但高風險須人工複核、廠商分多自報於策展子集）、綜合 5（量表 5-6 下緣）。三分皆與原評差 ≤1，無分歧（divergence=false）。

實證一：Benchmark（測項＋分數＋日期＋來源）

  • SpreadsheetBench（NeurIPS 2024；912 道源自 Excel 論壇的真實問題、2,729 測試檔，OJ 式多測資判分，涵蓋 find/extract/sum/highlight/modify/calculate 等 10 類含公式與圖表）：人類專家 71.33%（soft）；GPT-4o 18.35%、GPT-4＋SheetCopilot 14.00%、原生 Copilot in Excel 20%——SOTA 與人類差距逾 50 個百分點（來源1）。
  • SpreadsheetBench 後續榜（2025-2026）：Full（912）Copilot Agent Mode 57.2%（2025-09）、ChatGPT Agent w/ .xlsx 45.5%（裸 ChatGPT Agent 35.3%）、裸 Claude Files Opus 4.1 42.9%、o3 23.3%（多為自報）；Verified（400 題策展集）專用 agent 衝高：Decide 82.5%（2026-01，330/400）、GPT-for-Excel 92.5%、金山 Qingqiu 83–89%（Full 83.11／Verified 89.25）（來源2-4）。
  • SpreadsheetBench-V2（端到端商業流程，含財務建模/除錯/視覺化，多分頁、上千格修改）：最佳 Claude Opus 4.6 整體 34.89%（視覺化 62.5%、財模 34%、除錯僅 12%）、GPT-5.2 26.79%、Gemini 3.1 Pro 23.68%（2026-03，來源2）——難尾仍過半失敗。
  • SheetCopilot（NeurIPS 2023；28 工作簿、221 任務，含 Chart/Pivot/Formula 六類）：單次生成 Exec@1 44.3%，勝過強程式生成基線（來源5）。

實證二：真實部署/產品案例（誰用、規模、成效）

  • Microsoft Excel Copilot / Agent Mode：Copilot in Excel 與「Python in Excel」均已 GA，自然語言可生成 XLOOKUP/SUMIF 等公式、條件格式、樞紐與圖表；Agent Mode 2025-12 起 GA（rollout 至 2026-02），能自動建新分頁、寫公式、出樞紐/圖表並「迭代到驗證通過」（來源6-7）。內建於 M365，潛在用戶數以企業級計，但需訂閱授權與人工複核。
  • OpenAI ChatGPT（含 Agent / Code Interpreter）：可用 Python（openpyxl 等）產出帶公式與圖表的 .xlsx 供下載；SpreadsheetBench-Full：ChatGPT Agent w/ .xlsx 45.5%、裸 ChatGPT Agent 35.3%（來源2）。
  • Anthropic Claude（Files / Excel agent）：可生成可開啟的 .xlsx；裸模型 Verified 表現 42.9%，V2 視覺化類達 62.5%（來源2）。
  • 垂直新創：Decide（三人團隊、無創投，Verified 82.5%）、GPT-for-Excel（92.5%）、金山 WPS/Qingqiu、Shortcut.ai、Rows、Nobie 等多家以「Excel agent」為產品，公開榜競逐（來源2-4）。屬已商業化但多為小團隊/早期商用，缺大規模獨立成效披露。
  • 規模成效：通用助理（ChatGPT/Claude/Copilot）為數百萬級日用功能；但「無人把關自動產出對外財務試算表」仍少見，高風險場景普遍要求人工複核。

現況與邊界（AI 目前做得到/做不到）

  • 做得到：題意清楚的單表/中等複雜度任務——生成正確公式（非寫死值）、樞紐、標準圖表、條件格式，並交付可在 Excel 開啟的 .xlsx；專用 agent 在策展題集已逼近/超越人類專家（80–92%）。
  • 做不到/不穩：(1) 財務建模、除錯既有複雜活頁簿、跨分頁端到端流程仍弱（V2 最佳 ~35%、除錯 ~12%）；(2) 泛用裸模型在真實雜題只 ~20–43%，與人類專家落差大；(3) 會生出看似合理的錯誤公式、結果可重現性差；(4) 高分多為廠商自報於 Verified 子集，獨立驗證有限。
  • 邊界一句話：「依問題自動產出可開啟、公式正確的試算表」在中等任務已可靠且部分超越人類；複雜財模/除錯與可信交付仍需人工把關。
